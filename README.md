# Voice Analytics for Behavioral Scientists

This repository contains materials for the GSERM workshop **Voice Analytics for Behavioral Scientists: From Acoustic Features to Conversational AI Agents**.

The course introduces voice as a behavioral data source and walks through the full workflow: recording audio, preprocessing files, extracting acoustic features, working with speech-to-text, building simple conversational AI workflows, and evaluating validity, robustness, fairness, and research applications.

## Before the Course

There is nothing to prepare in advance beyond the technical setup:

- Bring a laptop.
- Install **R** and **RStudio**.
- Bring headphones or earphones.
- Make sure your laptop microphone works.

Readings, coding files, datasets, and exercises will be updated here and in the shared Dropbox folder before the course starts.

## Course Structure

- **Day 1:** Foundations, speech science, R setup, and first audio workflows.
- **Day 2:** Audio collection, device validity, consent, preprocessing, and open-science workflow.
- **Day 3:** Acoustic feature extraction, construct building, `voiceR`, modeling, and validation.
- **Day 4:** Speech-to-text, linguistic features, embeddings, and voice-agent toolbuilding.
- **Day 5:** Evaluation, fairness, robustness, experimental designs, and mini-project presentations.

## Software

The workshop is R-first. We will use packages such as `tuneR`, `seewave`, `soundgen`, `phonTools`, `voiceR`, `tidyverse`, `openai`, `googleLanguageR`, `talk`, and `reticulate`. Python may be used through R via `reticulate` for selected tasks, but no standalone Python workflow is required.
